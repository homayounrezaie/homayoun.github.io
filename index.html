5 Recent Projects

1 Agentic GeoAI Interface

brooksie.ai

Not launched yet - currently figuring out the business model.

Talk to Earth. An agentic interface where natural-language queries are routed by an orchestration layer to specialized tools - pulling live data from OpenStreetMap, Weather APIs, Google Earth Engine, GBIF, and other open geospatial sources - then processing it on the backend with GDAL, Rasterio, GeoPandas, and Shapely to answer in spatial context.

The agent planner decomposes each request, calls the right data sources and processing tools in sequence, and returns annotated evidence directly on the map - no manual GIS work required.

02 Thermography + AI

ThermoLabs.io

ThermoLabs.io is focused on turning thermal imagery into decision-ready intelligence. The project combines drone thermography with physics-aware machine learning, radiometric calibration, anomaly detection, and reporting workflows for real operational decisions.

The service direction includes drone data collection for methane emissions monitoring in oil and gas and landfill environments, plus thermal inspections for solar, buildings, powerlines, wind assets, aviation, and infrastructure. The goal is to move from raw thermal captures to calibrated orthomosaics, confidence-scored findings, zone-level statistics, interactive portals, and clear reports that teams can act on.

03 Deep Learning Pipeline

2D Floor Plans to 3D-BIM

During my master's thesis, I worked on 3D modeling for cities. With remote sensing data, we can often reach LOD2, which means we can model the outside shape of buildings. I built models using photogrammetry and LiDAR data, but I still wanted to reach LOD4, where indoor details also matter.

LOD4 is important for building energy modeling, but at that time creating it was honestly painful. Later, when I had some free time, I upgraded that workflow with OCR and deep learning models. The pipeline reads 2D architectural drawings, extracts plans, elevations, dimensions, walls, roofs, windows, doors, and geometry, then generates a 3D gbXML model for energy simulation and building performance analysis.

04 Spatial Interpolation Toolkit

geointerpo

geointerpo is a Python package for spatial interpolation, designed to make it easier to compare interpolation methods and understand how sparse point observations become continuous geospatial surfaces.

The package includes IDW, kriging, RBF, natural neighbor, cubic, and spline interpolation, with example notebooks that can be opened in Google Colab to reproduce the outputs. The repository, documentation, and PyPI package are linked below for implementation details and usage examples.

Research Lens

A lot of my research has been around thermal vision, especially midwave and longwave infrared data from drones, airborne surveys, and satellites. I built a reference list of satellites with thermal sensors onboard, and one pattern became clear: longwave thermal sensors still tend to have much coarser spatial resolution, while some midwave systems can reach meter-scale detail from space. It feels similar to computing, where CPUs and GPUs advanced very quickly while batteries moved more slowly. So the question I keep coming back to is: are we looking at an engineering problem, a physics limit, or the place where both meet?

How I Build GeoAI Systems

I think about GeoAI as the full data lifecycle — capturing data, processing it, storing it, training models, deploying them, and watching how they behave in production. Tools matter, but only when the workflow is clear. Once the steps are clear, picking the right tools is the easy part.

1. Data

Collecting data across platforms (drone, aircraft, satellite, ISS) and sensors (midwave + longwave thermal, LiDAR, hyperspectral, RGB), plus 3D data like point clouds and reconstructed meshes. I also fly drones and collect data myself.

Stack: STAC, COG, GeoPackage, GeoParquet, NetCDF, Zarr, GeoTIFF, GeoJSON, Shapefile, LAS/LAZ, metadata, OGC API.

2. Processing

Cleaning, preprocessing, and mosaicking raw imagery into something usable, radiometric calibration, georeferencing, structure-from-motion, and 3D reconstruction (orthomosaics, meshes, Gaussian splats).

Stack: GDAL, PDAL, rasterio, GeoPandas, NumPy, OpenCV, OpenDroneMap, Pix4D, QGIS, ArcGIS suite, Google Earth Engine.

3. Storage

Storing imagery, labels, and derived layers in databases and object storage so I can query them fast and version the training sets for reproducible experiments.

Stack: PostGIS, AWS S3, Cloud Optimized GeoTIFF tiling.

4. ML/DL Training & Testing

Picking model architectures, training them, and evaluating results. I work with vision-language models, transformers, autoencoders, knowledge distillation, geospatial foundation-model fine-tuning (SAM2, Prithvi, Clay), multimodal RAG, physics-informed ML, and time-series + spatial modeling.

Stack: PyTorch, scikit-learn, Hugging Face, MLflow, Weights & Biases, CUDA / multi-GPU, Jupyter, Python (venv / conda), Git, GitHub.

5. Deployment & Operations

Getting models out of notebooks and into production: packaging them in containers, setting up CI/CD, and exposing them as APIs other teams can call.

Stack: Docker, Kubernetes, FastAPI, AWS (SageMaker, Lambda), GitHub Actions.

6. Monitoring & Improvements

Watching how models perform once they're live, catching drift and edge cases, and retraining when the data shifts.

Stack: Evidently, scheduled retraining pipelines.
